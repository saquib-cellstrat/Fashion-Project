# Create Rule

## When to use

- The user asks for project-wide AI behavior rules.
- The user asks to formalize coding conventions in rule files.

## Required workflow

1. Identify scope (project or user-level) and intended trigger.
2. Write concise rules with clear do/don't statements.
3. Place rules in the appropriate project rule location.
4. Keep wording deterministic to reduce ambiguity.

## Quality checks

- Rules are actionable and specific.
- Rules avoid contradiction with existing project guidance.
- File references and examples are valid paths.
